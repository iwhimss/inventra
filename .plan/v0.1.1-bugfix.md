## Tespit Edilen Hatalar

### 🔴 Bug #1 — Login sayfası yüklenmiyor (VDS / Docker)

**Hata:**
```
FormatException: Invalid HTTP header field value: "/admin/login?error=Boş+bırakılamaz" (at character 22)
```

**Kök neden:**  
HTTP/1.1 standardı (RFC 7230), `Location` başlık değerlerinin yalnızca ASCII karakterler içermesini zorunlu kılar.  
`ş`, `ı`, `ğ`, `ü`, `ö`, `ç`, `İ` gibi Türkçe karakterler doğrudan Location header'a yazıldığında Shelf kütüphanesi `FormatException` fırlatır ve istek yanıtsız kalır — bu da login ekranının "yükleniyormuş gibi" görünmesine neden olur.

**Etkilenen satırlar (`admin_handler.dart`):**

| Satır | Mevcut (hatalı) | Türkçe char |
|-------|----------------|-------------|
| 121 | `Boş+bırakılamaz` | ş, ı |
| 128 | `Hatalı+bilgi` | ı |
| 134 | `Yetkiniz+yok` | — |
| 396 | `Kullanıcı+eklendi` | ı × 2 |
| 408 | `Kullanıcı+silindi` | ı × 2 |
| 571 | `Geçersiz+port+numarası` | ç, ı |
| 574 | `Geçersiz+host+değeri` | ç, ğ |
| 589 | `Yapılandırma+kaydedildi.+Sunucuyu+yeniden+başlatın.` | ı × 4, ş |
| 599 | `Onay+metni+hatalı.+Sıfırlama+iptal+edildi.` | ı × 3 |
| 741 | `Geçersiz+stok+değeri` | ç, ğ |
| 750 | `İzin+verilmeyen+alan` | İ |

**Çözüm:**  
`_redirect()` yanında `_buildUrl(String path, Map<String, String> query)` yardımcı fonksiyonu eklenecek.  
Bu fonksiyon `Uri.parse(path).replace(queryParameters: query).toString()` kullanarak tüm query parametre değerlerini otomatik olarak percent-encode eder.  
Tüm hardcoded Türkçe içerikli redirect URL'leri bu yardımcıya taşınacak.

### 🔴 Bug #2 — Ürünler sayfası Internal Server Error (her ortam)

**Hata:**
```
SqliteException(1): no such column: p.group_id, SQL logic error (code 1)
Causing statement: SELECT p.*, pg.name as group_name FROM products p
                   LEFT JOIN product_groups pg ON p.group_id = pg.id
                   ORDER BY p.name LIMIT 500
```

**Kök neden — 3 ayrı sorun:**

1. **Yanlış JOIN:** `products` tablosunda `group_id` sütunu yok. Tablo şeması `product_group TEXT` kullanıyor (grup adını doğrudan TEXT olarak saklar, FK değil).
2. **Yanlış kolon adı — okuma:** `p['price']` referansı var ama kolon adı `sale_price`.
3. **Yanlış kolon adı — yazma:** `UPDATE products SET price = ?` hatalı; doğrusu `sale_price`.

**Etkilenen satırlar (`admin_handler.dart`):**

| Satır | Mevcut (hatalı) | Düzeltme |
|-------|----------------|---------|
| 648 | `SELECT p.*, pg.name as group_name FROM products p LEFT JOIN product_groups pg ON p.group_id = pg.id` | JOIN kaldır, `SELECT * FROM products` kullan |
| 674 | `p['price']` | `p['sale_price']` |
| 682 | `p['group_name']` | `p['product_group']` |
| 747 | `UPDATE products SET price = ?` | `UPDATE products SET sale_price = ?` |

**Çözüm:**  
JOIN'i tamamen kaldır. `product_group` sütunu zaten grup adını string olarak tutuyor, ayrı sorguya gerek yok.  
Kolon adlarını veritabanı şemasıyla eşleştir.
