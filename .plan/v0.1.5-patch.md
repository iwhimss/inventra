## Kök Neden Analizi

### [B1] Kamera Siyah Alan

**Mevcut durum:**
```dart
MobileScanner(
  scanWindow: scanWindow,  // ← SORUN BURASI
  ...
)
```

**Sorunun mekanizması:**
`mobile_scanner` v6'da `scanWindow` parametresi iki şey yapıyor:
1. Barkod algılamayı yalnızca o `Rect` içinde yapıyor ✓
2. Kamera preview render'ını da o `Rect`'e kırpıyor ✗

Sonuç: Kamera görüntüsü yalnızca `scanWindow` Rect'i kadar render ediliyor, geri kalan ekran siyah kalıyor. Ekranda gördüğünüz siyah sol alan bu yüzden.

**Çözüm:**
- `scanWindow` parametresi `MobileScanner`'dan tamamen kaldırılır
- Kamera tüm ekranı doldurur (istenen davranış)
- Görsel overlay frame (`_ScanOverlay`) + animasyonlu scan çizgisi korunur (yalnızca kullanıcı yönlendirme amacıyla)
- Bölge filtresi için `barcode.corners` kullanılır — ANCAK bu veriler kamera koordinat sistemi (%), ekran koordinat sistemiyle çakışmıyor. `mobile_scanner` v6'da güvenilir koordinat dönüşümü için ek çalışma gerektirir
- **Pratik çözüm:** Kamera tam ekran, görsel frame yönlendirme için var, algılama bölgesi kısıtlaması için zaten mevcut olan **1.5s debounce** korunur. Bu debounce, alt alta barkodların kazara taranmasını önlemeye yeter.

### [B2] Klavye Kasma

**Sorunun mekanizması:**
Flutter'da varsayılan `Scaffold(resizeToAvoidBottomInset: true)` şu zinciri tetikler:
```
Klavye açılır
  → MediaQuery.viewInsets.bottom değişir  
  → MediaQuery'ye bağlı tüm widget'lar rebuilt
  → Scaffold constraint'leri değişir
  → Tüm içerik yeniden layout hesaplar + boyar
  → Büyük liste içeren ekranlarda jank
```

**Etkilenen ekranlar:**
- `pos_screen.dart`: Ürün listesi (`Consumer` + `ListView.builder`) — binlerce satır potansiyel
- `products_screen.dart`: Ürün listesi + tab içerikleri

**Çözüm Stratejisi:**
v0.1.5'te sepet bölümüne `RepaintBoundary` eklendi. Bu sürümde şunlar ekleniyor:

1. **`RepaintBoundary` ürün listesi üzerine** — listenin klavye animasyonu sırasında yeniden boyamasını engeller

2. **`resizeToAvoidBottomInset: false` + manuel padding** — Scaffold'un klavyede yeniden layout hesaplamasını engeller. Bunun yerine:
   ```dart
   Scaffold(resizeToAvoidBottomInset: false)
   // + en alttaki widget'a:
   Padding(
     padding: EdgeInsets.only(bottom: MediaQuery.viewInsetsOf(context).bottom),
   )
   ```
   Bu şekilde yalnızca en alt kısım (arama kutusu veya buton) yukarı kayar, liste yeniden hesaplanmaz.

3. **`MediaQuery.viewInsetsOf(context)` yerine `MediaQuery.of(context).viewInsets`** — Bu minor: sadece `viewInsets` değişince rebuild olan widget'ları izole eder, genel `MediaQuery` değişimlerini (brightness, textScaleFactor vb.) takip etmez.

**Önemli not:** `resizeToAvoidBottomInset: false` yapılırsa, floating search field veya alt kısımdaki butonlar klavye tarafından örtülebilir. Bu yüzden hangisinin etkilenebileceğini dikkatli analiz edip sadece gerekli yerlere padding eklenmeli.
