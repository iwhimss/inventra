## Kök Neden Analizi

### Bug #6 & #7 — Cache login çalışmıyor (en kritik sorun)

**Sorun:** "Beni Hatırla" seçilmesine rağmen uygulama yeniden açılınca giriş yapılamıyor. Sunucu offline iken de cache ile giriş yapılamıyor.

**Akış ve nerede bozuluyor:**

```
1. auth_provider.dart → login() başarılı (online) 
2. User.fromMap(response.data!['user']) → user nesnesi oluşturulur
3. user.toMap() şunu döner:
   {
     'id': ..., 'staff_id': ..., 'password_hash': ...,
     'role': ..., 'permissions': ...,
     'name': name ?? '',   ← SORUN BURADA
   }
4. db.insert('users', user.toMap()) çağrılır
5. client DB'nin users tablosunda 'name' kolonu YOK (CREATE TABLE'da eksik)
6. SqliteException: table users has no column named name
7. catch (_) {} ile sessizce yutuluyor
8. Kullanıcı ASLA cache'e kaydedilemiyor!
```

**Sonuç:** Her giriş denemesinde cache lookup başarısız olur çünkü kullanıcı hiç cache'e yazılamamıştır.

**İkincil sorun:** `/api/auth/login-hash` endpoint'i sunucuda yok. Auto-login sırasında 404 döner, offline cache'e düşer, ama cache de boş.

**Dosya:** `inventra_app/lib/core/database/database_helper.dart`

```dart
// MEVCUT (bozuk) — _createDB içinde:
await db.execute('''
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  staff_id TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  role TEXT NOT NULL,
  permissions TEXT     ← 'name' kolonu eksik!
)
''');

// DÜZELTME:
CREATE TABLE users (
  id TEXT PRIMARY KEY,
  staff_id TEXT NOT NULL UNIQUE,
  password_hash TEXT NOT NULL,
  role TEXT NOT NULL,
  name TEXT DEFAULT '',   ← eklenecek
  permissions TEXT
)
```

**Sunucu tarafı ek düzeltme:** `inventra_server/lib/core_server.dart`'a `/api/auth/login-hash` endpoint'i ekle. Bu endpoint `staff_id` + `password_hash` alır, doğrudan hash karşılaştırması yapar, kullanıcı bilgisini döner. Böylece auto-login hash ile sunucuya da doğrulanabilir.

### Bug #8 — Açık tema: Converter sayfası renk sorunları

**Sorun 1 — Başlık görünmüyor:**
```dart
// converter_screen.dart - Scaffold backgroundColor hardcoded dark
Scaffold(backgroundColor: AppTheme.darkBackground,  // Açık temada da koyu!
```
Başlık metni `Theme.of(context).textTheme.headlineSmall` rengi → ışık temasında koyu mavi. Arka plan da light mode'da açık. Görünmeli aslında. 

Ama `Text('Excel Sütun Eşleştirici', style: Theme.of(context).textTheme.headlineSmall?.copyWith(fontWeight: FontWeight.bold))` - Bu theme'den renk alıyor. Scaffold background `AppTheme.darkBackground` ise, light modda bu `lightBg` (açık gri). ThemeData'nın textTheme rengi de açık modda koyu olacak. Dolayısıyla görünmeli...

Asıl sorun muhtemelen şu: `AppTheme.darkBackground` statik bir değer ve başlatıldığında `lightBg`. Ama `setDarkMode` çağrıldığında güncelleniyor. Scaffold build sırasında state doğru mu?

**Sorun 2 — Dropdown içindeki yazılar görünmüyor:**
```dart
// _buildDropdown içinde:
dropdownColor: AppTheme.darkBackground,   // açık temada açık arka plan
items: [
  DropdownMenuItem(child: Text('Yok / Boş Bırak', style: TextStyle(fontSize: 12))),  
  // TextStyle explicit color yok → theme default → açık temada koyu
]
```

Bu aslında çalışmalı... Ama sorun şu: `DropdownMenuItem` içindeki `Text` için tema rengi düzgün uygulanmıyor olabilir. `dropdownColor` açık arka plan → metin de açık → görünmüyor.

**Düzeltme:** Dropdown item metinlerine `color: AppTheme.textMain` ekle. `dropdownColor` için `AppTheme.panelBackground` kullan (temaya uyumlu).

## Notlar

- **DB migration:** `users` tablosuna `name` eklenmesi için version bump **gerekmiyor** — v16 migration'ı zaten `ALTER TABLE users ADD COLUMN name TEXT DEFAULT ''` yapıyor. Sadece `_createDB`'deki CREATE TABLE güncellenir (yeni kurulumlar için).
- **Muhtelif Ürün:** Satış kaydında `product_id = 'misc_...'` olan kayıtlar normal ürün değildir; stok düşülmez, raporda "Muhtelif" olarak gösterilmelidir. Sunucu zaten bilinmeyen product_id'leri hata üretmeden kaydediyor.
- **Sunucu switch:** `navigatorKey` `main.dart`'ta tanımlı, tüm yerlerden erişilebilir.
