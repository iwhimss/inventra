### Bug #1 — "Güvenli değil" + Yanlış subdomain içeriği

**Tetikleyici:** v0.1.1'de verilen talimat: *"port 443 bloğunu sil, sadece port 80 kalsın"* + *"Cloudflare'i Full moduna al"*

**Akış:**
```
Tarayıcı → HTTPS → Cloudflare (Full mod)
Cloudflare → port 443 → VDS Nginx
Nginx: inventra.fatihdikec.me için port 443 bloğu YOK
Nginx: default_server'a düşüyor → jrwhims.fatihdikec.me bloğu
Nginx: jrwhims sertifikasını sunuyor
Tarayıcı: domain uyuşmuyor → "güvenli değil"
```

**Neden önceki talimat yanlıştı:**  
Cloudflare "Full" SSL modu, origin sunucusunda geçerli bir SSL sertifikası ve port 443 bloğu gerektirir.  
Port 443 bloğu kaldırıldığında Nginx bu bağlantıyı bilemez ve yanlış server block'a yönlendirir.

---

### Doğru Çözüm: Cloudflare "Flexible" Modu

Elde iki seçenek var:

| | Flexible (Önerilen) | Full |
|---|---|---|
| Cloudflare → VDS | HTTP (port 80) | HTTPS (port 443) |
| VDS'de SSL cert gerekir mi? | ❌ Hayır | ✅ Evet (Certbot) |
| Nginx config | Sadece port 80 | Port 80 + 443 |
| Karmaşıklık | Düşük | Orta |
| Güvenlik | Cloudflare edge'de HTTPS | Uçtan uca HTTPS |

**Inventra için Flexible yeterli ve önerilir** — zaten yalnızca Cloudflare üzerinden erişildiği için edge HTTPS koruması yeterli.

**Flexible mod akışı:**
```
Tarayıcı → HTTPS → Cloudflare (SSL bitimi Cloudflare'de)
Cloudflare → HTTP (port 80) → VDS Nginx
Nginx → HTTP → Docker (port 5000)
```

Tarayıcı HTTPS görür (Cloudflare sağlar), Nginx'te SSL cert gerekmez.

### B) Nginx konfigürasyonunu kontrol et

Mevcut `/etc/nginx/sites-available/inventra` dosyasında şu yapı olmalı:

```nginx
map $http_upgrade $connection_upgrade {
    default upgrade;
    ''      close;
}

server {
    listen 80;
    server_name inventra.fatihdikec.me;

    location / {
        proxy_pass         http://127.0.0.1:5000;
        proxy_http_version 1.1;
        proxy_set_header   Upgrade           $http_upgrade;
        proxy_set_header   Connection        $connection_upgrade;
        proxy_set_header   Host              $host;
        proxy_set_header   X-Real-IP         $remote_addr;
        proxy_set_header   X-Forwarded-For   $proxy_add_x_forwarded_for;
        proxy_set_header   X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }
}
```

**Port 443 bloğu OLMAMALI** (Flexible modda Cloudflare port 80'e bağlanır, 443 gerekmez).

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## Notlar

- Certbot daha önce port 443 bloğu eklemiş olabilir. `sudo nano /etc/nginx/sites-available/inventra` ile kontrol et ve `server { listen 443 ssl; ... }` bloğunu sil.
- Certbot'un otomatik değiştirdiği config bazen `/etc/nginx/sites-enabled/inventra` içindedir, `/etc/nginx/sites-available/inventra` ile symlink'tir. İkisini de kontrol et.
- `jrwhims.fatihdikec.me` bloğundaki `default_server` işaretini kaldırmanı öneririm (seni etkileyen sorunun bir parçası bu olabilir).
