## Kök Neden Analizi

### [B1] Kamera Siyah Alan — 2. Analiz

**Önceki yaklaşım:** `scanWindow` parametresini kaldırdık → HATALI. Siyah alan hâlâ devam ediyor.

**Gerçek kök neden:**
`Stack` widget'ının `fit` parametresi varsayılan olarak `StackFit.loose`'dur. Bu durumda Stack, çocuklarına "istedikleri kadar büyüyebilirsiniz" der ama zorunlu tutmaz. `MobileScanner` widget'ı kendi aspect ratio'suna göre boyutlanabilir (ör: 4:3 kamera → ekran 9:20 → preview solda siyah bırakır).

**Delil:** Kırmızı ile işaretlenen siyah alan çerçevenin SOL kenarında — kamera preview'i tam genişliğe yayılmadığında, kalanı siyah.

**Çözüm:**
```dart
// Önce:
Stack(
  children: [
    MobileScanner(...),  // kendi boyutunda
    ...
  ],
)

// Sonra:
Stack(
  fit: StackFit.expand,   // TÜM çocukları Stack boyutuna zorlar
  children: [
    MobileScanner(
      fit: BoxFit.cover,   // Preview ekranı kaplasın, kırp
      ...
    ),
    ...
  ],
)
```

`StackFit.expand` → tüm çocuklar Stack'in tam boyutuna expand edilir (Scaffold body = tam ekran).  
`BoxFit.cover` → kamera preview aspect ratio'ya göre merkezi kırpılır, siyah bar kalmaz.

### [B2] Klavye Kasma — 2. Analiz

**Önceki yaklaşım:** `RepaintBoundary` eklendi → KISMI. Sadece repaint maliyetini düşürüyor, layout'u durdurmuyordu.

**Gerçek kök neden:**
`MediaQuery.of(context)` → build metodunda çağrıldığında, widget **tüm** MediaQuery değişikliklerinde rebuild tetikliyor. Klavye animasyonu sırasında `MediaQuery.viewInsets.bottom` 0'dan ~300dp'ye 60fps'de çıkıyor → ~18 kare boyunca her karede tüm ekran rebuild.

```
Klavye animasyonu (300ms @ 60fps = ~18 kare)
  → Her karede MediaQuery.viewInsets değişir
  → MediaQuery.of(context) kullanan widget'lar rebuild
  → POS build() → product section build → ListView layout → 18× hesaplama
  → Jank / FPS düşüşü
```

**Tespit edilen sorunlu satırlar:**
- `pos_screen.dart:524` → `MediaQuery.of(context).size.width > 800`
- `products_screen.dart:399` → `MediaQuery.of(context).size.width > 800 ? ...`

**Çözüm:** Flutter 3.7+ targeted MediaQuery accessors

```dart
// Klavyede rebuild tetikler:
MediaQuery.of(context).size.width

// Sadece ekran boyutu değişince rebuild eder, klavyede ETKİLENMEZ:
MediaQuery.sizeOf(context).width
```

`MediaQuery.sizeOf(context)` yalnızca `size` değiştiğinde rebuild tetikler.
`viewInsets` (klavye) değiştiğinde tetiklemez.

Flutter SDK requirement: `^3.11.0` → mevcut. `sizeOf()` Flutter 3.7'de eklendi, uyumlu.

**Ek iyileştirme:** `barcode_scanner_page.dart:62`'de de aynı dönüşüm yapılacak.
