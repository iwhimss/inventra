## Kök Neden Analizleri

### Sorun 1: "Server başlatınca direkt çalışıyor, instance nerede belli değil"

`server.dart` başlarken sadece sunucu logları basar, data dizininin tam yolunu yazmaz.  
`serverBaseDir` mantığı: compile edilmişse exe'nin bulunduğu klasör, `dart run` ile çalışıyorsa CWD. Bu sessizce işler, kullanıcı nerede olduğunu bilemez.

**Çözüm:** Sunucu başlangıç mesajına `Data dizini: <path>` satırı ekle.

### Sorun 2: "Sıfırdan oluşturmak istersem?"

`setup.dart`'ta zaten mevcut config varsa üzerine yazma sorusu soruluyor. Ancak `--setup` bayrağı bilinmiyorsa sıfırlama yolu belirsiz.

**Çözüm:** `--reset` bayrağı ekle (`data/config.json` + `data/inventra.db`'yi sil → setup wizard başlat). Admin panelden de "Kurulumu Sıfırla" aksiyonu ekle (onay ekranı ile).

### Sorun 3: "Ayarlarını düzenlemek istersem?"

Port/host değişikliği için tek yol şu an: `--port`/`--host` CLI parametreleri veya `config.json`'u manuel düzenleme.

**Çözüm:** Admin paneli Settings sayfasına sunucu yapılandırması bölümü ekle. Port ve host web arayüzünden değiştirilebilsin. Değişiklik `config.json`'a yazılsın, yeniden başlatma gerektiği belirtilsin.

### Sorun 4: "Web Dashboard işlevsiz"

Mevcut: Dashboard (istatistik), Cihazlar, Kullanıcılar, Roller, Ayarlar.  
Eksik: Ürün yönetimi, satış geçmişi, aktivite logları, stok uyarıları, sunucu config yönetimi.

### Sorun 5: "Terminal gözükmesin, ama erişilebilsin"

`dart run` geliştirici modunda terminali açık tutar, derlenmemiş çalışır. Üretimde `.exe`'ye derlenip NSSM ile Windows servisi olarak kayıt yapılmalı. NSSM logları dosyaya yazar; log dosyası takibiyle terminalden erişim sağlanabilir.

### 🔲 FAZA 4 — Docker Desteği

**Hedef:** inventra_server'ı Docker container'ında çalıştır. Veri kaybı yaşanmasın (volume). Ortam değişkeniyle ilk kurulum otomatikleştirilebilsin.

#### 4A — Dockerfile (`inventra_server/Dockerfile`)

Multi-stage build:
```
Stage 1 (dart:stable): dart pub get + dart compile exe → /app/inventra_server
Stage 2 (debian:stable-slim): libsqlite3-0 kur, binary kopyala, VOLUME /app/data, EXPOSE 5000
```

Entrypoint: İlk başlangıçta `data/config.json` yoksa ENV değişkenlerine bak:
- `INVENTRA_BUSINESS_NAME`, `INVENTRA_ADMIN_ID`, `INVENTRA_ADMIN_PASSWORD`, `INVENTRA_PORT` (default: 5000)
- Varsa → interaktif sormadan otomatik setup yap
- Yoksa → "docker exec -it inventra-server ./inventra_server --setup komutunu çalıştırın" yazar ve bekler

Bu ENV desteği için `setup.dart`'a yeni `runNonInteractive()` metodu eklenecek.

- [x] **4.1** `inventra_server/Dockerfile` oluştur (multi-stage)
- [x] **4.2** `inventra_server/.dockerignore` oluştur (`.dart_tool/`, `data/`, `*.db`)
- [x] **4.3** `setup.dart`'a `SetupWizard.runFromEnv()` metodu ekle (ENV'den okur)
- [x] **4.4** `bin/server.dart` giriş noktasına Docker ENV kontrolü ekle (config yoksa + ENV varsa auto-setup)

#### 4B — docker-compose.yml (proje kök dizini)

```yaml
services:
  inventra-server:
    build: ./inventra_server
    container_name: inventra-server
    restart: unless-stopped
    ports:
      - "5000:5000"
    volumes:
      - inventra_data:/app/data
    environment:
      - TZ=Europe/Istanbul
      # İlk kurulum için (sonra kaldırılabilir):
      # - INVENTRA_SETUP=true
      # - INVENTRA_BUSINESS_NAME=Mağazam
      # - INVENTRA_ADMIN_ID=1000
      # - INVENTRA_ADMIN_PASSWORD=güvenli_şifre
      # - INVENTRA_PORT=5000
volumes:
  inventra_data:
```

## Soru / Açık Noktalar

> **Windows Service NSSM vs Task Scheduler?**  
> Plan NSSM ile tasarlandı. NSSM üçüncü parti ama çok yaygın ve güvenilir (crash'te otomatik yeniden başlatma, servis yönetimi). Alternatif olarak Windows Task Scheduler (built-in) kullanılabilir — ama gerçek servis davranışı yoktur. Onayda tercihini belirt.

> **Dashboard Ürünler sayfası: Sadece görüntüleme mi, düzenleme de olsun mu?**  
> Plan stok düzenleme (inline sayı girişi) içeriyor. Ürün fiyatı/adı düzenleme de isteniyor mu? Yoksa sadece stok yönetimi yeterli mi?

> **Docker Setup ENV değişkenleri zorunlu mu?**  
> Plan, `docker-compose.yml`'de ENV değişkenlerini yorum satırı olarak bırakıyor. İlk kurulum için `docker exec -it inventra-server ./inventra_server --setup` interaktif komutunu çalıştırma seçeneği de geçerli. İkisini de destekleyeceğiz.
