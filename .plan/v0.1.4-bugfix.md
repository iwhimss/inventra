### Bug #4 — Ayarlar > "Geçiş" Butonu Çalışmıyor

**Kök Neden:**

```dart
// sync_settings_tab.dart — _switchServer() içinde:
final db = await DatabaseHelper.instance.database;  // ← YANLIŞ! Per-server cache DB
await db.insert('settings', {'key': 'server_ip', 'value': url}, ...);
// Ardından SystemNavigator.pop() veya exit(0) çağrılıyor
```

`DatabaseHelper.instance.database` → o anki sunucunun cache DB'si (örn. `inventra_cache_192_168_1_100.db`).  
Uygulama yeniden başladığında `_init()` → `globalDb`'den `server_ip` okuyor → eski sunucu adresi görünüyor → geçiş olmuyor.

**Dosya:** `inventra_app/lib/features/settings/tabs/sync_settings_tab.dart`

**Çözüm:**
1. `DatabaseHelper.instance.database` → `DatabaseHelper.instance.globalDb` (kritik düzeltme)
2. Uygulamayı kapatmak yerine **in-app switch**:
   ```
   globalDb'ye server_ip kaydet
   → DatabaseHelper.switchToServer(url)
   → syncProvider.connectToServer(url)
   → authProvider.logout()
   → pair durumuna göre: LoginScreen veya ServerConnectScreen
   ```

**Not:** Confirmation dialog'unu koruyabiliriz ama "DEĞİŞTİR VE ÇIK" → "DEĞİŞTİR" olarak güncelle.

### Bug #5 — Sunucu Ekleme Anında Çıkış Yapıyor

**Sorun:** `_showAddServerDialog()` başarılı bağlantıdan sonra otomatik olarak:
```dart
await ref.read(authProvider.notifier).logout();
navigatorKey.currentState?.pushAndRemoveUntil(LoginScreen(), ...)
```
yaparak kullanıcıyı anında çıkartıyor. Kullanıcı henüz geçiş yapmak istemeyebilir, sunucuyu sadece listeye eklemek istemiş olabilir.

**Dosya:** `inventra_app/lib/features/settings/tabs/sync_settings_tab.dart`

**Çözüm:** `_showAddServerDialog()` sadece:
1. Bağlantı test et (sunucu erişilebilir mi kontrol et)
2. Listeye ekle (`saved_servers`)
3. Başarı mesajı göster: "Sunucu eklendi. Geçiş yapmak için listedeki 'Geçiş' butonunu kullanın."
4. Auto-logout / auto-navigate YOK

**Dikkat:** Bağlantı testi için `connectToServer()` çağırmak yerine sadece `ApiClient.isOnline()` benzeri basit bir health check yeterli. `connectToServer()` çağrılırsa mevcut bağlantı bozulabilir.

Yeni yaklaşım:
```dart
// Sadece server'ın erişilebilir olduğunu doğrula:
ApiClient.instance.configure(baseUrl: ip);
final isReachable = await ApiClient.instance.isOnline();
// Eski sunucuyu geri yükle (geçiş henüz istemiyoruz):
ApiClient.instance.configure(baseUrl: currentActiveUrl);

if (isReachable) {
  await _addSavedServer(ip);
  showSuccessSnackbar('Sunucu eklendi. Geçiş için "Geçiş" butonunu kullanın.');
} else {
  // Sunucuya ulaşılamadı ama yine de ekle (belki offline sunucu olsun diye)
  await _addSavedServer(ip);
  showWarningSnackbar('Sunucu eklendi ancak şu an erişilemiyor.');
}
```

### Bug #6 — Eşleşme Onayı Gereksiz Tekrar Gönderiliyor

**Sorun:** `connectToServer()` her çağrıldığında `requestPairing()` yapar. Bu:
1. Sunucu zaten bu cihazı `approved` olarak bilse bile yeni bir pairing request gönderir
2. Sunucu bunun üstesinden gelir (zaten pair varsa approved döndürür) ama gereksiz istek
3. Asıl sorun: `pair_status` global DB'de saklanıyor ve sunucu bağımsız — farklı sunucuların status'ları karışıyor (Bug #7 ile bağlantılı)

**Ek sorun:** Uygulama restart edildiğinde `_init()` globalDb'den `pair_status = 'approved'` okuyabilir (önceki sunucudan), AppGate bunu görüp `LoginScreen` gösterebilir, sonra health check gerçek durumu öğrenince `pairStatus` değişir ve kullanıcı `ServerConnectScreen`'e düşer — kötü UX.

**Dosya:** `inventra_app/lib/features/pos/providers/sync_provider.dart`

**Çözüm — `connectToServer()` içinde:**
```dart
// Pairing request göndermeden önce status kontrol et:
final existingStatus = await ApiClient.instance.checkPairStatus(deviceId);
if (existingStatus['status'] == 'approved') {
  // Zaten onaylı, gereksiz pairing request gönderme
  final apiKey = existingStatus['api_key'] as String?;
  if (apiKey != null) { /* save api_key */ }
  state = state.copyWith(isOnline: true, pairStatus: 'approved', serverUrl: url);
  _startHealthCheck();
  return 'approved';
}
// Değilse normal pairing request akışı devam eder
```

### Bug #7 — Cache Sistemi Sunucular Arası İzole Değil

**Mevcut durum:**
- Cache DB: ✅ Per-server (`inventra_cache_{sanitized_url}.db`) — zaten izole
- `globalDb` ayarları: ❌ Global (`pair_status`, `api_key` tek anahtar) — karışıyor

**Sorun:** `pair_status = 'approved'` ve `api_key` globalDb'de tek anahtar olarak saklanıyor.
- Server A'ya bağlanılıp onaylandıktan sonra `pair_status = 'approved'`
- Server B'ye geçildiğinde uygulama globalDb'den `pair_status = 'approved'` okur
- Ama bu Server B için geçerli değil
- Health check sonunda düzelir ama başlangıçta yanlış yönlendirme olur

**Dosya:** `inventra_app/lib/features/pos/providers/sync_provider.dart`

**Çözüm:** `pair_status` ve `api_key` için sunucu-spesifik anahtar:

```dart
String _serverKey(String url) =>
    url.replaceAll(RegExp(r'[^a-zA-Z0-9]'), '_');

// Okuma:
'pair_status_${_serverKey(url)}'
'api_key_${_serverKey(url)}'

// Yazma: aynı şekilde
```

**Etkilenen yerler:**
- `sync_provider._init()` — okuma
- `sync_provider.connectToServer()` — yazma
- `sync_provider.checkConnection()` — yazma
- `sync_provider.disconnectFromServer()` — silme

**Geriye dönük uyumluluk:** İlk çalışmada eski `pair_status` anahtarını okuyup yeni anahtara taşı, eskiyi sil.

## Teknik Notlar

### In-app Server Switch Akışı
```
Kullanıcı "Geçiş" tıklar
→ Confirmation dialog
→ globalDb'ye server_ip yaz (BUG FIX: globalDb, database değil)
→ DatabaseHelper.switchToServer(url)
→ syncProvider.connectToServer(url) [yeni pair check + approved döner]
→ authProvider.logout()
→ pair == 'approved' → navigatorKey.pushAndRemoveUntil(LoginScreen)
→ pair == 'pending'  → navigatorKey.pushAndRemoveUntil(ServerConnectScreen)
→ pair == 'offline'  → hata göster, eski sunucuya geri dön
```

### Cache Anahtar Şeması
```
globalDb ayarları (shared / server-independent):
  server_ip          → aktif sunucu
  saved_servers      → sunucu listesi
  device_id          → cihaz kimliği (tüm sunucular için aynı)
  remember_me        → beni hatırla bayrağı
  saved_staff_id     → kaydedilen kullanıcı ID
  saved_password_hash → hash

globalDb ayarları (per-server):
  pair_status_{key}  → bu sunucu için eşleşme durumu
  api_key_{key}      → bu sunucu için API anahtarı

_serverKey(url) = url.replaceAll(RegExp(r'[^a-zA-Z0-9]'), '_')
// Örn: "192.168.1.100:5000" → "192_168_1_100_5000"
```

### Sunucu Ekleme vs. Sunucu Geçiş (Ayrımı)
- **Ekle:** Sadece listeye ekler. Mevcut bağlantı değişmez.
- **Geçiş:** Aktif sunucuyu değiştirir, login gerektirir, pair kontrol eder.
- **Login ekranı:** Listeden seçip geçiş yapar (login sayfasından çıkmadan).
