---

## Kök Neden Analizi

### CMake Cache Hatası
Rework sırasında `merchant_app/` → `inventra_app/` dizin yeniden adlandırması yapıldı.  
Ancak `inventra_app/build/windows/x64/CMakeCache.txt` dosyası eski path'i (`merchant/merchant_app/build/...`) içeriyor.  
CMake, mevcut kaynak path'i (`inventra/inventra_app/...`) ile cache'deki path'i karşılaştırıyor, uyumsuzluk bulunca build'i reddediyor.

**Çözüm:** `build/` klasörünü tamamen silmek. Flutter sonraki `flutter run` veya `flutter build` çağrısında bu klasörü sıfırdan oluşturur.

### Hardcoded Versiyon Değerleri
Sunucu tarafında `/api/version` endpoint'inde versiyon stringleri elle yazılmış:
- `core_server.dart:2106` → `'server_version': '1.0.0'`
- `core_server.dart:2108` → `'min_app_version': '1.0.0'`

Uygulama tarafında fallback değer güncel değil:
- `app_settings_tab.dart:44` → `String _appVersion = 'v1.0.0'`

## Fazlar

---

### 🔲 FAZA 0 — CMake Cache Temizliği

**Hedef:** Stale build klasörünü silerek CMake'in yeni path ile temiz başlamasını sağlamak.

- [x] **0.1** `inventra_app/build/` klasörünü tamamen sil (`flutter clean` ile)
  > `build/` klasörü yalnızca derlenmiş çıktıları barındırır, kaynak koda ait hiçbir şey içermez.  
  > `.gitignore`'da zaten hariç tutulmuş durumda — silmek güvenlidir.

**Doğrulama:**
```powershell
cd inventra_app
flutter run -d windows
```
CMake hatasının kaybolması bekleniyor.

---

### 🔲 FAZA 1 — Versiyon Bilgisi Güncellemesi

**Hedef:** Tüm hardcoded versiyon stringlerini `0.0.1` ile eşleştirmek.

#### 1.1 — Sunucu Tarafı (`inventra_server`)

- [x] **1.1.1** `inventra_server/lib/core_server.dart:2106`
  ```dart
  // Önce:
  'server_version': '1.0.0',
  // Sonra:
  'server_version': '0.0.1',
  ```

- [x] **1.1.2** `inventra_server/lib/core_server.dart:2108`
  ```dart
  // Önce:
  'min_app_version': '1.0.0',
  // Sonra:
  'min_app_version': '0.0.1',
  ```

> **Not:** `min_app_version`, sunucunun "bu sürümden düşük app'lere izin verme" eşiği.
> Şu an her iki taraf da `0.0.1` olduğu için eşit → güncelleme dialog'u tetiklenmez.

#### 1.2 — Uygulama Tarafı (`inventra_app`)

- [x] **1.2.1** `inventra_app/lib/features/settings/tabs/app_settings_tab.dart:44`
  ```dart
  // Önce:
  String _appVersion = 'v1.0.0';
  // Sonra:
  String _appVersion = 'v0.0.1';
  ```
  > Bu bir fallback değeri; gerçek versiyon `PackageInfo.fromPlatform()` ile `pubspec.yaml`'dan okunuyor.
  > pubspec.yaml zaten `0.0.1+1` — yalnızca fallback güncelleniyor.
